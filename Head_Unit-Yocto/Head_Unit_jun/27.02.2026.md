## Fixes & Features Added Today

### 1. Ambient Light — Signal Connection Bug Fix
**Problem:**
`ambient/main.cpp` was connecting `AmbientService::colorChanged` and `AmbientService::lightingOff` signals to the shell bridge. However, `AmbientScreen` (the UI widget) never calls `AmbientService` — it emits `ambientColorChanged` and `ambientOff` signals directly. As a result, color changes made in the UI were never forwarded to the shell, so the ambient glow overlay never updated.

**Fix:**
- Changed connections in `modules/ambient/main.cpp` to connect `AmbientScreen` (window) signals to the bridge instead of `AmbientService` signals.
- Also fixed a type mismatch: `AmbientScreen::ambientColorChanged` emits `int brightness` but `ShellClient::sendAmbientColor` expects `quint8 brightness` — resolved using a lambda wrapper.

**Files changed:**
- `modules/ambient/main.cpp`

### 3. Gamepad Gear → Head Unit Sync
**Problem:**
When the gamepad changes gear (X=Drive, B=Reverse, Y/A=Neutral), it goes through `piracer_bridge.py` which writes directly to `/tmp/piracer_drive_mode.json`. The Head Unit had no mechanism to read this file, so the GearPanel and all module displays never reflected gamepad-initiated gear changes.

**Root Cause:**
The gear flow was one-directional:
- HU touch → VSOMEIP → IC → motor ✓
- Gamepad → `piracer_bridge.py` → file → motor (HU unaware) ✗

**Fix:**
Added a 500ms polling timer (`pollGamepadGear()`) in `ShellWindow` that:
1. Reads `/tmp/piracer_drive_mode.json`
2. Checks file freshness (ignores files older than 3 seconds to avoid stale data)
3. Maps direction string to `GearState`: `"F"` → D, `"R"` → R, `"N"` → N
4. Applies smart filtering to prevent echo loops:
   - Skips if file direction unchanged from last read (`m_lastFileGearDir`)
   - Skips if current gear already maps to the same direction (prevents overwriting P with N — both are direction "N" in the file)
5. Calls `m_gearStateManager->setGear(gear, "gamepad")` which triggers `broadcastGearState()` → all module GearPanels update.

**Files changed:**
- `shell/ShellWindow.h` — added `pollGamepadGear()` slot, `m_lastFileGearDir`, `m_gearFilePollTimer`
- `shell/ShellWindow.cpp` — added includes (`QFile`, `QFileInfo`, `QJsonDocument`, `QDateTime`), timer setup in `setupConnections()`, `pollGamepadGear()` and `gearToDirection()` implementations
